# Patakbuhin ang halimbawa

## I-install ang mga kinakailangan

```sh
npm install
```

## I-build ito

```sh
npm run build
```

## Patakbuhin ito

```sh
npm start
```

Makikita mo ang text na:

```text
Registering tools...
Starting server...
```

Magaling, ibig sabihin tama ang pagsisimula nito.

## Subukan ang server

Subukan ang mga kakayahan gamit ang sumusunod na command:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Dapat nitong simulan ang web interface ng inspector tool.

### Subukan sa CLI mode

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Patakbuhin ang isang tool:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Makikita mo ang tugon na katulad ng:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Subukang subukan ang isang tool na hindi umiiral tulad ng "add2" gamit ang command na ito:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Makikita mo na ngayon ang mensaheng ito na nagpapakita na gumagana ang iyong validation:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Subukan din magpadala ng parameter na `c` na dapat tanggihan ng schema tulad nito:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Makikita mo na ngayon ang "invalid arguments" error:

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
