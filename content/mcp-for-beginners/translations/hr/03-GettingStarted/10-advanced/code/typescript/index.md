# Pokreni primjer

## Instaliraj ovisnosti

```sh
npm install
```

## Izgradi

```sh
npm run build
```

## Pokreni

```sh
npm start
```

Trebali biste vidjeti tekst:

```text
Registering tools...
Starting server...
```

Odlično, to znači da se ispravno pokreće.

## Testiraj server

Isprobajte funkcionalnosti sljedećom naredbom:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Ovo bi trebalo pokrenuti web sučelje alata za inspekciju.

### Testiraj u CLI načinu

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Pokreni alat:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Trebali biste vidjeti odgovor sličan:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Pokušajte testirati alat koji ne postoji, poput "add2", koristeći ovu naredbu:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Sada biste trebali vidjeti ovu poruku koja pokazuje da vaša validacija radi:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Također pokušajte poslati parametar `c` koji bi trebao biti odbijen prema shemi, ovako:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Sada biste trebali vidjeti grešku "neispravni argumenti":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
