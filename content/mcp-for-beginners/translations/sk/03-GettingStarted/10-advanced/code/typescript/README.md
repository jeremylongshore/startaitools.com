# Spustenie ukážky

## Inštalácia závislostí

```sh
npm install
```

## Zostavenie

```sh
npm run build
```

## Spustenie

```sh
npm start
```

Mali by ste vidieť text:

```text
Registering tools...
Starting server...
```

Skvelé, to znamená, že sa to správne spúšťa.

## Testovanie servera

Otestujte funkcie pomocou nasledujúceho príkazu:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Tým by sa mala spustiť webová rozhranie nástroja inspector.

Spustite nástroj:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Mali by ste vidieť odpoveď podobnú:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Skúste otestovať nástroj, ktorý neexistuje, napríklad "add2", pomocou tohto príkazu:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Teraz by ste mali vidieť túto správu, ktorá ukazuje, že vaša validácia funguje:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Skúste tiež poslať parameter `c`, ktorý by mal byť odmietnutý schémou, napríklad takto:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Teraz by ste mali vidieť chybu "neplatné argumenty":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
