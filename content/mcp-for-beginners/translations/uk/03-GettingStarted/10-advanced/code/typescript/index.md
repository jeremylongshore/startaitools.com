# Запуск прикладу

## Встановіть залежності

```sh
npm install
```

## Зберіть проект

```sh
npm run build
```

## Запустіть його

```sh
npm start
```

Ви повинні побачити текст:

```text
Registering tools...
Starting server...
```

Чудово, це означає, що він запускається правильно.

## Тестування сервера

Перевірте можливості за допомогою наступної команди:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Це має запустити веб-інтерфейс інструменту інспектора.

### Тестування в режимі CLI

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Ви повинні побачити наступний результат:

Запустіть інструмент:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Ви повинні побачити відповідь, схожу на:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Спробуйте протестувати інструмент, який не існує, наприклад "add2", за допомогою цієї команди:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Тепер ви повинні побачити це повідомлення, яке показує, що ваша валідація працює:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Також спробуйте надіслати параметр `c`, який має бути відхилений схемою, наприклад:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Тепер ви повинні побачити помилку "недійсні аргументи":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
