# Запуск примера

## Установка зависимостей

```sh
npm install
```

## Сборка

```sh
npm run build
```

## Запуск

```sh
npm start
```

Вы должны увидеть текст:

```text
Registering tools...
Starting server...
```

Отлично, это значит, что всё запускается правильно.

## Тестирование сервера

Проверьте возможности с помощью следующей команды:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Это должно запустить веб-интерфейс инструмента инспектора.

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Запуск инструмента:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Вы должны увидеть ответ, похожий на:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Попробуйте протестировать инструмент, который не существует, например "add2", с помощью этой команды:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Теперь вы должны увидеть сообщение, показывающее, что ваша проверка работает:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Также попробуйте отправить параметр `c`, который должен быть отклонён схемой, например так:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Теперь вы должны увидеть ошибку "недопустимые аргументы":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
