# Spuštění ukázky

## Instalace závislostí

```sh
npm install
```

## Sestavení

```sh
npm run build
```

## Spuštění

```sh
npm start
```

Měli byste vidět text:

```text
Registering tools...
Starting server...
```

Skvělé, to znamená, že se to správně spouští.

## Testování serveru

Otestujte funkce pomocí následujícího příkazu:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Tím by se mělo spustit webové rozhraní inspektorového nástroje.

Spuštění nástroje:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Měli byste vidět odpověď podobnou:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Zkuste otestovat nástroj, který neexistuje, například "add2", pomocí tohoto příkazu:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Nyní byste měli vidět tuto zprávu, která ukazuje, že vaše validace funguje:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Také zkuste poslat parametr `c`, který by měl být odmítnut schématem, například takto:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Nyní byste měli vidět chybu "neplatné argumenty":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
