# Executar exemplo

## Instalar dependências

```sh
npm install
```

## Construir

```sh
npm run build
```

## Executar

```sh
npm start
```

Deverá ver o texto:

```text
Registering tools...
Starting server...
```

Ótimo, isso significa que está a iniciar corretamente.

## Testar o servidor

Teste as funcionalidades com o seguinte comando:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Isto deverá iniciar a interface web da ferramenta de inspeção.

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Executar uma ferramenta:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Deverá ver uma resposta semelhante a:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Experimente testar uma ferramenta que não existe, como "add2", com este comando:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Agora deverá ver esta mensagem indicando que a validação funciona:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Também experimente enviar um parâmetro `c` que deverá ser rejeitado pelo esquema, como este:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Agora deverá ver o erro "argumentos inválidos":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
