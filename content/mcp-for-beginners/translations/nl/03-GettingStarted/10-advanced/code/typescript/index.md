# Voorbeeld uitvoeren

## Installeer afhankelijkheden

```sh
npm install
```

## Bouw het

```sh
npm run build
```

## Voer het uit

```sh
npm start
```

Je zou de tekst moeten zien:

```text
Registering tools...
Starting server...
```

Geweldig, dat betekent dat het correct opstart.

## Test de server

Test de mogelijkheden met het volgende commando:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Dit zou de webinterface van de inspectietool moeten starten.

npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list

Voer een tool uit:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Je zou een reactie moeten zien die lijkt op:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Probeer een tool te testen die niet bestaat, zoals "add2", met dit commando:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Je zou nu dit bericht moeten zien dat aangeeft dat je validatie werkt:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Probeer ook een parameter `c` te sturen die door het schema afgewezen zou moeten worden, zoals:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Je zou nu de foutmelding "ongeldige argumenten" moeten zien:

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
