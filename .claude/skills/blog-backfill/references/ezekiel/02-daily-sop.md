# Ezekiel Daily SOP — Standing Runbook

**This is your everyday checklist.** Setup is done once (see `01-onboarding-setup.md`). This doc is what you run every day.

**The one rule that never changes:** The packet email tells you exactly what to post and where. You copy, paste, and post. You do **not** write the copy — it's provided, pre-written, in the right voice for each platform. Your job is accurate, complete, on-time execution.

---

## Daily Flow at a Glance

1. Open the **posting packet** email in `ezekiel@intentsolutions.io`.
2. Read any **"before posting" note / disclaimer** at the top — those are verbatim-required (see below).
3. Check the packet's **destination list** for today's post (it tells you the tier / which platforms apply).
4. Post to each applicable destination using the steps below.
5. Set **canonical URL** on Substack + Medium where applicable.
6. Put **links in the first comment** on LinkedIn (both personal and company).
7. **Log what you posted** (reply to the packet email with the resulting URLs).

If **no packet arrives**, that is a signal — see Escalation. Silence is never normal.

## Per-Platform Posting Steps

### X / Twitter

1. Switch into Jeremy's account (delegate).
2. Paste the packet's **X post (#1)** and publish.
3. **If the packet includes a thread or carousel** (used on bigger posts): post it as a **thread** — post #1 first, then reply to it with #2, reply to #2 with #3, and so on, in order. Keep the sequence exactly as the packet lists it.
4. If the packet gives a scheduled time and you have X Pro scheduling, schedule it; otherwise just post it live.

- [ ] X post published (thread in order if provided)

### LinkedIn — personal profile (via Buffer)

1. In Buffer, compose to Jeremy's **personal** LinkedIn channel.
2. Paste the packet's **LinkedIn personal** copy into the post **body**. Do **not** put the link in the body.
3. Publish / schedule.
4. As soon as it's live, paste the packet's **"first comment" block** as the **first comment** on the post — this is where the link goes.

- [ ] Personal post published
- [ ] Link posted in the **first comment** (not the body)

### LinkedIn — Intent Solutions company page (NATIVE, not Buffer)

1. Post **directly in LinkedIn** as the company page (you have Content Admin). **Do not use Buffer for the company page.**
2. Paste the packet's **LinkedIn company** copy into the post **body**. No link in the body.
3. Publish.
4. Immediately paste the **"first comment" block** as the **first comment** — link goes here.

- [ ] Company post published natively
- [ ] Link posted in the **first comment** (not the body)

**Why links go in the first comment:** LinkedIn's algorithm suppresses reach on posts with links in the body. Putting the link in the first comment keeps reach up. The packet gives you a ready first-comment block for exactly this — always use it.

### Substack (Tier 2/3 only)

1. Create a new Substack post.
2. Paste the **long-form article body** — copy it from the **live startaitools.com page** (the packet gives you the URL).
3. Go to **Settings → Canonical URL** and paste the **exact startaitools.com URL from the packet, verbatim**. This is **REQUIRED** — it tells search engines the original startaitools post is the source, so the Substack copy doesn't outrank the original.
4. Publish.

- [ ] Body pasted from the live page
- [ ] Canonical URL set to the exact startaitools.com URL (verbatim)

### Medium (Tier 2/3 only)

**Preferred method — Import (Medium auto-sets canonical for you):**

1. **Avatar → Write → ⋯ (more) → Import a story.**
2. Paste the **startaitools.com URL** (from the packet) → **Import**.
3. Medium pulls in the article **and auto-sets the canonical link** to the original. Review, then publish.

**Fallback method — if Import isn't working:**

1. Copy-paste the article body from the live startaitools.com page.
2. Set canonical manually: **⋯ → Settings → Advanced → Customize canonical link** → paste the exact startaitools.com URL from the packet.
3. Publish.

- [ ] Imported (or pasted) from the live page
- [ ] Canonical link points to the exact startaitools.com URL

## The Verbatim-Disclaimer Rule (non-negotiable)

Some packets include a **"before posting" editorial note** or a **required disclaimer string** (for example, "keep the tone fair to Anthropic").

- Copy any such disclaimer text **exactly** into the post where the packet says to put it.
- **Do not edit it. Do not shorten it. Do not omit it. Do not paraphrase it.**
- If a disclaimer is required, it goes on **every** destination the packet says it applies to.
- If a disclaimer is unclear or you're not sure where it goes → **do not guess. Escalate** (see below).

A day's post is **done** only when all of these are true:

- [ ] Posted to **every** destination the packet lists for that tier.
- [ ] LinkedIn (personal **and** company): link is in the **first comment**, not the body.

## Escalation Runbook

Ping Jeremy (email + whatever fast channel he prefers) in any of these cases. Don't guess, don't silently skip.

| Situation | What to do |
|---|---|
| **No packet arrived** | Silence is never normal. If the day's packet isn't in your inbox by your normal posting time, ping Jeremy. Note: a "NO PACKET TODAY — <reason>" message is itself the signal — if you got that, no action needed, but confirm you saw it. |
| **Packet looks wrong** (missing sections, garbled copy, wrong tier) | Don't post. Ping Jeremy with a screenshot / paste of what's off. |
| **A link in the packet 404s** | Don't post a dead link. Ping Jeremy; wait for a corrected URL. |
| **A disclaimer is unclear** (unsure of wording or placement) | Do not guess or paraphrase. Ping Jeremy and confirm exact text + location before posting. |
| **A platform is down / you can't publish** | Post to the destinations that work, log which one failed, ping Jeremy so it can be re-tried. |

### Backup poster

If you're unavailable (sick, traveling, offline), the **backup poster** covers the day: **Pablo or Max**. Hand off the packet (forward the email) and confirm they've got it so a day doesn't get skipped. A missed day does **not** auto-catch-up — someone has to post it, so don't leave it silent.
