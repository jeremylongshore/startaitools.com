# Ezekiel Onboarding — One-Time Platform Setup

**Who this is for:** Ezekiel (`ezekiel@intentsolutions.io`), taking over all social-media posting for Intent Solutions / Jeremy Longshore's blog (startaitools.com).

**What you'll do here:** Get access to every platform and tool you need, once. After this checklist is done, you never touch it again — your daily job is the runbook in `02-daily-sop.md`.

**How the job works in one paragraph:** Every day the blog publishes a post to startaitools.com. An automated "posting packet" email lands in your inbox (`ezekiel@intentsolutions.io`, CC Jeremy). That packet contains ready-to-paste copy for each platform — you copy, paste, and post. You do **not** write the copy. It's already written for you. This setup doc just gets you the keys.

## 2. X / Twitter — Delegate access (Contributor role)

You post to Jeremy's X account as a **delegate** — you never need his password.

**Jeremy does this (account owner side):**

1. X account → **Settings and privacy** → **Security and account access** → **Delegate** → **Manage members**.
2. Add `Ezekiel` (his X handle) as a member with the **Contributor** role.
3. Ezekiel accepts the invite from his own X account.

**Ezekiel does this (your side):**

- [ ] Accept the delegate invite in your own X account.
- [ ] Confirm you can switch into Jeremy's account and see the compose box.
- [ ] Post one throwaway test (a period, then delete it) to confirm you can publish.

**Scheduling — verify at onboarding:**

- X Pro (formerly TweetDeck) scheduling requires **X Premium+** on the account.
- [ ] **Jeremy: verify whether a Contributor delegate inherits Pro scheduling access.** If yes, Ezekiel schedules through X Pro (`pro.x.com`).
- If a delegate does **not** inherit Pro scheduling, the fallback is simple: Ezekiel posts via the **standard delegate access** at `pro.x.com` or `x.com` — i.e., just post it live when the packet arrives, no scheduling needed.

**SKIP the paid X API.** Do not set up or pay for the X API. At ~$0.20/post it is not worth it for simple link/text posts. Delegate posting covers everything we do.

## 3. LinkedIn — Jeremy's personal profile (via Buffer Free)

You schedule to Jeremy's **personal** LinkedIn profile using **Buffer Free**.

- [ ] Create (or log into) the Buffer account Jeremy designates (Jeremy provides login).
- [ ] Buffer Free allows **1 connected channel** — connect **Jeremy's personal LinkedIn profile** as that one channel.
- [ ] Confirm you can compose and schedule a post to the personal profile from Buffer.

**Why Buffer here:** Buffer Free is fine for the single personal-profile channel and lets you schedule ahead. That's all we need it for.

## 4. LinkedIn — Intent Solutions company page (Content Admin, NATIVE posting)

The company page is handled **differently** from the personal profile. Read this carefully.

**Jeremy does this (grant access):**

1. Go to the Intent Solutions LinkedIn company page → **Admin tools** → **Manage admins**.
2. **Add admin** → search for Ezekiel → select the **Content Admin** role.
3. Send the invite. **The invite expires in 30 days — Ezekiel must accept promptly.**

**Ezekiel does this (accept + verify):**

- [ ] Accept the Content Admin invite **within 30 days** (don't let it lapse).
- [ ] Confirm you can access the company page and see the "post as page" option.

**IMPORTANT — do NOT use Buffer for the company page.**

- Buffer requires **Super Admin** to connect a company page.
- We intentionally give you only **Content Admin** (narrower, safer access).
- Therefore the company page is posted to **NATIVELY** — directly inside LinkedIn, not through Buffer.
- Your daily runbook reflects this: personal profile → Buffer; company page → native LinkedIn.
